# projects
This project analyzes pizza restaurant sales data using SQL to identify customer ordering patterns, top-performing pizzas, revenue contribution, and business trends. The goal is to transform raw transactional data into actionable insights for menu optimization, inventory planning, and revenue growth.

Objective:

Analyze pizza sales performance

Identify most ordered pizza sizes

Find top-selling pizza types

Measure revenue contribution by category

Understand customer ordering behavior by time

Dataset Information

This project contains 4 relational tables:

1. order_details

Columns:

order_details_id

order_id

pizza_id

quantity

Rows: 48,620

2. orders

Columns:

order_id

date
time

Rows: 21,350

3. pizza_types

Columns:

pizza_type_id

name

category

ingredients

Rows: 32


4. pizzas

Columns:

pizza_id

pizza_type_id

size

price

Rows: 96

Business Insights:

Popular pizza sizes indicate customer buying preference

Top pizzas drive major sales volume

Peak order hours support staffing decisions

Revenue leaders help menu promotion

Category performance supports inventory planning

Conclusion:

This SQL analysis reveals actionable business insights for restaurant growth through product performance tracking, revenue analysis, and customer behavior understanding. Regular data analysis can improve operational efficiency and menu strategy.
